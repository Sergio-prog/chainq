## Findings considered and rejected

- **Watch/stream mode + threshold alerts**: a daemon changes the tool's character and agents can loop the CLI themselves; stays on ROADMAP "Later". Not planned.
- **MCP wrapper**: deliberately parked pending demand from non-shell agents (ROADMAP "Later"); the skill-first thesis is working. Not planned.
- **CEX prices via ccxt**: heavy dependency for a cross-check CoinGecko mostly covers. Not planned.
- **Robinhood Chain testnet and stock-token registry**: mainnet support is the
  useful zero-setup primitive; testnet does not match the production registry,
  and token addresses need separate official validation. Not included in 007.
- **Kamino's entire product suite in one integration**: liquidity strategies,
  KVaults, leverage, transactions, and raw user obligations have different
  schemas and accounting risk. Plan 008 deliberately ships lending markets;
  normalized wallet positions are the recommended follow-up.
- **Pump trading and transaction builders**: signing, slippage, fee routing,
  simulations, and fast-changing account requirements are a materially
  different security surface. Plan 009 remains read-only.
- **Top-level `chainq read` command**: v0.16 independently shipped generic
  ABI-less contract reads as `chainq evm call`, backed by shared calldata and
  ABI decoding helpers. A second parser and top-level alias would duplicate the
  same capability without adding material coverage. Plan 002 is rejected.
